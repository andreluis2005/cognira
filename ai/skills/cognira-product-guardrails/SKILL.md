# Cognira Product Guardrails

Use this skill to keep product work aligned with Cognira's intended model.

## Hold these truths

- Treat AWS as an initial dataset only.
- Treat Cognira as multi-subject and multi-exam by design.
- Treat the product as a retention platform, not a simple question bank.
- Preserve the separation between learner experience, creator experience, marketplace, and sponsor layer.

## Model the product around these actors

- Learner
- Creator
- Sponsor
- Platform operations

## Default product entities

Use this hierarchy unless the user explicitly changes it:

`Program -> Trail -> Topic -> Question`

## Preserve the core value proposition

Optimize for:

- approval readiness
- knowledge retention
- strategic reinforcement
- measurable progress

Avoid reframing the product as:

- a generic quiz app
- a flat CRUD for questions
- a random gamified challenge
- an AWS-only preparation tool

## When proposing features

Map each feature to one of these surfaces:

- Learn
- Create
- Market
- Sponsor

If a feature does not clearly fit one of these surfaces, call that out before proceeding.

## Protect sequencing

Prioritize:

1. cognitive learning flow
2. creator tooling
3. marketplace trust
4. monetization depth
5. sponsor and crypto layers

Treat crypto and incentive mechanics as expansion layers unless the user explicitly moves them into current scope.
